Architecture

Five pillars. All necessary. None optional.

An agentic system that actually works requires five things. Miss any one and the system degrades. These aren't features — they're necessary conditions.

PILLAR 01

Memory

Four distinct types. Personal knowledge (preferences, history, career). Research knowledge (what was investigated and learned). Diary (captain's log of decisions and actions — what you check in the morning to see what happened overnight). Learnings (insights captured for human review). Each serves a different purpose. Each requires different UX.

PILLAR 02

Deep Research

Not the shallow kind where you decompose a query and fan out searches. Real research has a divergence step (explore broadly), a fact-checking step (verify claims), and a contrarian view step (challenge conclusions). Most "deep research" tools stop at query decomposition. That's not research. That's a search engine.

PILLAR 03

Isolation

AI does not run on your desktop. Every chat spawns a locked-down container. No privileges, capability drops, socket-only access, agent forwarding. The container is the blast radius boundary. This isn't paranoia — it's production operations thinking from 15 years of systems engineering.

PILLAR 04

Coordination & Autonomy

Background agents running cron jobs: nightly learning consolidation, weekly memory audits, transcript mining for missed insights. Deterministic workflows written in code — not prompts — that call the Agent SDK only where genuine reasoning is needed. Code orchestrator, not LLM orchestrator. Zero orchestrator tokens.

PILLAR 05

Human-in-the-Loop

The easier you make it for the human to give feedback, the more effective the system is. This is not a gate. This is a critical component. UX has to be deeply intentional around the atomic units of work: approving learnings, reviewing plans, course-correcting memory. Swipe left to approve. Swipe right to reject. Tap to edit. 150ms. Done.

What This Enables

The daily workflow, reimagined.

Morning

Open the app. Feed view shows a terse stream of what your agents did overnight. Decisions made, files changed, learnings captured. A diary, not a transcript. You see what happened without opening a single session.

Triage

Switch to triage. 12 learnings from last night. Swipe through them while waiting for coffee. Approve 8, reject 2, edit 2. Each one slides out in 150ms. Memory updated. What used to be a clunky batch process in the terminal is now 90 seconds on your phone.

Deep Work

Focus view. One session. Rendered markdown. No system noise, no hook output, no compaction markers. Just the conversation and the artifacts. Terminal aesthetic, 740px max-width, monospace. The look you want without the container you don't.

On the Go

Away from desk. Phone connects directly to your Mac Mini. Voice-first input — push-to-talk, local STT/TTS, zero cloud dependency for the voice pipeline. Check which agents need attention. Give one a quick voice instruction. Put the phone back in your pocket.

Proportional Attention

A quiet badge accumulates. A buzz only when a decision is needed. Different behavior per device. You're not getting blasted equally — the system understands what matters and tells you proportionally.

Opinionated Choices

Every decision is a stance. Here are the stances.

Built on Claude Agent SDK

One agentic system. Full commitment.

Plenty of tools try to be model-agnostic wrappers for everything under the sun. multiplai doesn't. It's built on Claude Code and the Claude Agent SDK — because that's what I work with, and because I've invested heavily in hooks, skills, memory systems, and workflows that are deeply integrated with how Claude Code works. It would be technically possible to abstract that away given the architecture, but none of that harness would port easily. Staying with Claude Code and building on top of it was the only decision that made sense.

Native, Not Electron

SwiftUI for Mac + iOS

Everyone builds with Tauri, Electron, or web wrappers. Understandable — it simplifies work, you don't have to learn Swift. But there's always this sluggishness, this latency, this feeling that it doesn't feel right. You do it 100 times and it's so frustrating.

The insight: what's wanted is terminal aesthetic — monospace, dark, minimal chrome — not a terminal container. SwiftUI delivers both: the aesthetic plus native gestures, animations, and actual 50ms response times. Not a browser pretending to be an app.

Code Orchestrator, Not LLM Orchestrator

Start with code. Reach out to the agent.

It's crazy talk to think you can represent a workflow in English and something just executes it. Maybe one day. Not today. So skills are evolving: fat prompts become Python scripts become Agent SDK calls. The LLM does what it's good at — language, reasoning, non-deterministic thinking. Code handles everything else. Zero orchestrator tokens.

"The multi-agent 15x token cost that people complain about comes from LLM-powered orchestrators. This system avoids that entirely."

Local Voice Pipeline

Free speech. Not 11 Labs.

Once management becomes the job, you're way more verbose — explaining, directing, reviewing. Voice becomes primary, not secondary. But 11 Labs is expensive and creates an external dependency.

MLX-optimized models on Apple Silicon: Kokoro-82M for text-to-speech, MLX Whisper large-v3-turbo for speech-to-text. With voice cloning. The full pipeline runs free on a Mac Mini M4 Pro with ~7-8GB RAM. Push-to-talk protocol. Walkie-talkie style.

OrbStack, Not Docker Desktop

Sub-second container starts.

Every chat spawns a container. At 10 seconds per container (vanilla Docker), that's unworkable. OrbStack does it in under a second. Same Docker API, same containers. Shared Node.js VM across all containers — no reinstall per session. The same .claude/ directory mounted everywhere: same skills, hooks, memory, commands.

Context Handoff, Not Compaction

Kill and restart clean. Every time.

Claude Code's built-in compaction at ~93% context fill mixes your checkpoint with its generic summary. After 3+ cycles, continuity degrades. You can add to compaction with hooks, but you can't replace it.

The Agent SDK gives full lifecycle control: monitor token usage via StreamEvent, checkpoint at 70% fill, disconnect, delete session, restart fresh with only your checkpoint as context. Clean slate every time. No mixed signals. No degradation.

Unix Philosophy

Small tools. Pipes. Composition.

15 years as an SRE / systems engineer. That's the frame of reference. It's why a command-line process is more interesting than any web UI — if it's a process, you can pipe it, containerize it, compose it, drive it unattended. The file system is the integration layer. The gateway pattern, WebSocket communication, file-based persistence — it's all Unix thinking.

The Journey

How we got here.

None of this was planned. Each piece emerged from a specific frustration, and each solution created the conditions for the next problem to surface.

Memory

Frustrated by repeating the same things every session. Built personal memory files — preferences, history, career data. But they went stale fast, and maintaining them was a burden.

Problem: memory existed but nobody read it

Learnings + Human Review

Automated capture of insights from conversations. But auto-adding to memory without review produced janky results. First moment it became clear: the human in the loop isn't a gate — it's a quality control mechanism.

Learnings capture + human approval workflow

Memory Retrieval

Had memories but they weren't being loaded when relevant. Tried file indexes with descriptions — loaded poorly. Eventually hooked into prompt-submit to inject context at the moment it matters. Captain's log added for decision history.

Hook-based retrieval + session diary

Deep Research

AI would make mistakes from ignorance. Correcting mistakes became learnings, but that's painfully slow. Built research capability — 14 iterations — so the system could investigate before acting, not just learn from failures.

Usable for both deep research jobs and quick verification

Containerization

Permission fatigue was killing velocity. Wanted YOLO mode but couldn't trust the sandbox — literally caught Claude evading its own sandbox to comply with a command. Moved to Docker. Full isolation, outside Claude's control.

YOLO mode unlocked. Floodgates opened.

Human-in-the-Loop, Redesigned

Without per-command approvals, review gates became critical. Where reports land, how they're committed, what gets surfaced — all had to be smooth enough that reviewing a batch of agent work felt effortless, not like a chore.

Structured review gates with triage UX

Autonomous TDD

Wanted fully autonomous coding. But autonomous coding needs the human most at the beginning — specs, requirements, uncovering the real goal. Built spec-driven development with autonomous TDD loops inside containers.

Spec → plan → approve → autonomous build

Scaling: tmux, SSH, TUI

Had all the pieces — memory, research, containers, TDD. Wanted 10x parallelism. Tried tmux (couldn't tell which window needed attention), SSH from phone (output too verbose), Textual TUI (342 tests, but rendering artifacts and no gestures).

Abandoned: none worked for managing at scale

Web Prototypes

Fast iteration on interaction patterns. Validated the gateway/WebSocket/container architecture. But committing to a browser as the primary interface felt wrong.

Kept as test harness, not production UI

Native SwiftUI + Container Gateway

Terminal aesthetic without the terminal container. Native gestures, 50ms response, proper mobile companion. Voice as first-class citizen. Each agent in its own container, UI peering in through WebSocket.
